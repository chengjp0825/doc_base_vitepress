# 社区投稿与质量规范 📝

为了保证“硬件笔记分享站”的内容专业、易读且长久可用，我们制定了以下简单的投稿指南。欢迎每一位硬核开发者参与共建！

## 1. 基本排版要求

- **中文排版**：使用中文全角标点，中文与英文/数字之间建议保留一个空格。
- **标题层级**：从 `##` (H2) 开始使用二级标题，严禁全文只有一个 `<h1>` 标题。
- **文件名**：一律使用小写字母标识，如 `stm32g0.md`，不要使用空格。

## 2. 内容组件规范

为了提升阅读体验，请善用 VuePress 提供的容器组件：

- **提示 (Tip)**：用于分享设计灵感、小技巧。
  ```markdown
  ::: tip 设计小贴士
  这里可以写一些你发现的巧妙电路连接方式。
  :::
  ```
- **警告 (Warning)**：用于提醒可能导致调试困难的注意事项。
- **危险 (Danger)**：**必看！** 用于标注容易导致烧毁硬件、炸管子或产生不可逆后果的代码/操作。

## 3. 代码块标准

所有代码块必须标注语言类型，并包含必要的注释。

```c
// 推荐的 C 语言注释风格：清晰描述寄存器操作目的
RCC->IOPENR |= RCC_IOPENR_GPIOAEN; // 开启 GPIOA 时钟
```

## 4. 图表与公式

- **原理图**：推荐使用 Mermaid 流程图描述简单的信号流，或者上传清晰的截图。
- **计算公式**：涉及电阻分压、滤波频率等计算，请使用 LaTeX 格式：
  - 示例：`$f_c = \frac{1}{2\pi RC}$`

## 5. 如何提交？

1. **Fork** 本仓库。
2. 在 `docs/` 对应目录下参考 `TEMPLATE.md` 创建你的笔记。
3. 提交 **Pull Request**。我们会尽快进行 Review 并合并！
