# 预加重与去加重：高速链路里的边沿补偿策略

在高速数字链路中，通道通常像一个低通系统：频率越高，衰减越明显。结果就是边沿变钝、眼图闭合、误码率上升。

预加重（Pre-emphasis）和去加重（De-emphasis）就是为这个问题服务的两种发送端均衡手段。

---

## 1. 两者到底在做什么

- 预加重：在比特翻转瞬间，短时间提高高频能量，让边沿更陡。
- 去加重：在发送持续同电平时，适当降低后续幅度，突出过渡信息。

工程上可以把它们理解为同一类“发送端频谱整形”策略，不同芯片命名会有差异。

---

## 2. 为什么能改善眼图

长走线、连接器、过孔和介质损耗会优先吞掉高频分量。补偿后，接收端看到的过渡边沿会更清晰。

简化理解：

1. 通道损耗高频。
2. 发送端提前“补高频”。
3. 接收端得到更接近理想波形的结果。

---

## 3. 什么时候该用

以下场景通常优先考虑预加重/去加重：

- 速率上升后眼图裕量明显变差。
- 板间或背板链路较长。
- 通道插损较高（连接器、线缆、过孔较多）。

::: warning 注意
预加重/去加重并不是越大越好。参数过激会导致过冲、抖动和 EMI 变差。
:::

---

## 4. 调参思路（实战）

推荐流程：

1. 固定接收端均衡参数，先扫发送端预加重/去加重档位。
2. 以眼高、眼宽和 BER 作为主要评价指标。
3. 选择满足目标 BER 的最小补偿档位，避免过补偿。
